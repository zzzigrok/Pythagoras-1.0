## 1. Где находятся гиперпараметры в коде?

Если вы используете единый центр управления проектом, откройте файл `pythagoras_hub.py` (или соответствующие файлы скриптов обучения) и найдите в самом верху блок, озаглавленный `# --- 1. ГЛОБАЛЬНЫЕ НАСТРОЙКИ ---`.

```python
# Гиперпараметры архитектуры Pythagoras (v2.0)
n_embd = 256       # Размерность вектора для одного символа (ширина)
n_head = 8         # Количество параллельных "голов" внимания
n_layer = 6        # Глубина размышлений (количество блоков Трансформера подряд)
block_size = 64    # Контекстное окно (максимальная длина строки в символах)
dropout = 0.1      # Вероятность "забывания" (регуляризация)
```

## 2. Детальный разбор ключевых параметров

### 🧠 `n_embd` (Embedding Dimension)
**Что это:** Ширина скрытого состояния модели (Dimensionality). Сколько чисел (в виде длинного вектора) модель выделяет для описания математического смысла одного единственного символа.
**Влияние:** Это самый "тяжелый" параметр. Увеличение с `256` до `512` сделает вашу модель теоретически в 4 раза умнее (так как размер матриц линейных слоев увеличивается квадратично: $512 \times 512$), но она будет требовать радикально больше видеопамяти (VRAM) для работы и обучения.

### 👀 `n_head` (Number of Attention Heads)
**Что это:** На сколько логических частей мы делим механизм внимания (Multi-Head Attention).
**Влияние:** При базовых настройках `n_head = 8` и `n_embd = 256`, каждая "голова" имеет размерность $256 / 8 = 32$. Головы работают параллельно, каждая фокусируясь на своей задаче. Одна голова может выучить следить только за знаками операций (`+` или `-`), другая голова — за десятками, третья — искать условия для переноса единицы. 
> [!WARNING]
> *Важное математическое правило: `n_embd` должен нацело делиться на `n_head`, иначе PyTorch выдаст ошибку инициализации.*

### 🏢 `n_layer` (Number of Transformer Blocks)
**Что это:** Глубина нейросети. Сколько раз вектор информации прогоняется через конвейер (Блок Внимания -> Блок Мышления / FFN).
**Влияние:** Слишком мало слоев (например, `2`) — модель останется поверхностной и не сможет выучить сложную многошаговую логику, такую как заимствование разрядов при вычитании. Слишком много (например, `24`) — обучение будет очень медленным, а модель подвержена переобучению. `6` слоев — классический, проверенный временем стандарт для небольших моделей (GPT-1/GPT-2 style).

### 📏 `block_size` (Context Window)
**Что это:** Окно внимания. Максимальное количество символов, которое модель способна "видеть" одновременно.
**Влияние:** Базовый `block_size = 64`. Если мы решаем пример `999+999=1998\n`, то его длина всего 14 символов. Окна в 64 хватит с огромным запасом. Увеличивать этот параметр стоит **только** если вы модифицировали `prep_math.py` для генерации сверхдлинных выражений (например, `120+450-10+88=`) или уравнений.

## 3. Сценарии: Как подобрать идеальные значения?

В зависимости от вашего железа, вы можете менять код прямо в `pythagoras_hub.py`:

**🐢 Сценарий 1: Micro-Pythagoras (Для слабых ПК на базе CPU)**
Если у вас нет видеокарты, или вы запускаете код на старом ноутбуке:
```python
n_embd = 128
n_head = 4
n_layer = 4
```
*Эффект: Модель будет обучаться быстро даже на процессоре, потреблять минимум RAM, но её интеллектуальный потолок будет низким (ошибки на сложных 3-значных примерах).*

**🚀 Сценарий 2: Архи-Трансформер (Для мощных GPU типа RTX 4090 / 3090)**
Если в вашем распоряжении есть 16-24 ГБ VRAM, позвольте модели стать монстром:
```python
n_embd = 512
n_head = 16
n_layer = 12
```
*Эффект: Модель выучит сложнейшие математические правила за считанные минуты и выдаст точность близкую к 100%. Однако, итоговый файл весов `.pth` будет весить сотни мегабайт или гигабайты.*

## 4. Критические правила безопасности

> [!CRITICAL]
> **Несовместимость весов (Shape Mismatch)**
> Нейросеть — это набор жестко заданных матриц (таблиц с числами). Файл `math_model_weights.pth` хранит именно эти таблицы. 
> Если вы обучили модель с `n_embd = 256`, а затем в коде поменяли его на `n_embd = 512`, при следующем запуске (чата или тренировки) **PyTorch выдаст фатальную ошибку (Shape Mismatch Error)**, так как попытается загрузить квадратную матрицу $256\times256$ в место, отведенное под $512\times512$.
> 
> **Единственное решение:** Перед любым изменением гиперпараметров (`n_embd`, `n_head`, `n_layer`, `block_size`), вы **ОБЯЗАНЫ** удалить (или переименовать) старый файл `math_model_weights.pth` и запустить процесс обучения с полного нуля!

## 5. Глоссарий Гиперпараметров

| Термин | Значение |
| :--- | :--- |
| **Гиперпараметры (Hyperparameters)** | Настройки модели, которые задаются разработчиком ДО начала обучения (в отличие от обычных "весов" или "параметров", которые модель учит сама). |
| **Dropout** | Техника борьбы с переобучением (зубрежкой). При `dropout = 0.1` на каждом шаге обучения 10% случайных нейронов "отключаются", заставляя оставшиеся 90% учить общие правила арифметики, а не просто запоминать конкретные примеры. |
| **Shape Mismatch** | Ошибка PyTorch, возникающая при попытке загрузить веса одной архитектуры (например, 6 слоев) в модель с другой архитектурой (например, 12 слоев). |
