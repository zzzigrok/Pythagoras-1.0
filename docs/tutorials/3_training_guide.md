> [!TIP]
> Обучение Трансформера — это потрясающий, но ресурсоемкий процесс. ## 1. Скрипты для обучения (UI vs Console)

В проекте Pythagoras 1.0 предусмотрено несколько способов запустить тренировку, адаптированных под разные среды:
1. **`train_math.py`** — Базовый, сырой скрипт. Выводит логи простой стеной текста прямо в консоль. Идеально подходит для запуска на удаленных Linux-серверах через SSH или в Docker-контейнерах, где нет поддержки сложной графики терминала.
2. **`train_math_rich.py`** — Скрипт с красивым UI. Использует библиотеку `rich` для отрисовки динамических прогресс-баров, цветных табличек и статусов.
3. **`pythagoras_hub.py`** — Центральный хаб (рекомендуется!). Просто запустите его, выберите пункт **«1. ТРЕНИРОВКА МОДЕЛИ»**, и он автоматически поднимет красивый интерфейс для обучения.

## 2. Выбор оборудования (CPU, GPU, XPU)

Нейросети, особенно Трансформеры, требуют гигантского количества одновременных матричных умножений. Выбор устройства для тренировки критически важен:

*   **CPU (Центральный процессор):** Работает всегда и везде, но **очень медленно**. Обучение модели до приемлемого уровня на CPU может занять сутки. Подходит только для проверки того, что код запускается и не падает с ошибками.
*   **GPU (Видеокарта NVIDIA / CUDA):** Отраслевой стандарт. Ускоряет обучение в десятки раз за счет тысяч потоковых процессоров.
*   **XPU (Видеокарта Intel Arc / Iris):** Pythagoras 1.0 имеет встроенную поддержку `intel-extension-for-pytorch`, что позволяет владельцам видеокарт Intel (от встроенной графики до дискретных Arc A770) обучать модели аппаратно и с высокой скоростью!

В коде хаба (`pythagoras_hub.py`) устройство определяется автоматически. Если нейросеть найдет видеокарту, она сама перенесет все данные в ее память (VRAM).
```python
# Автоопределение в коде:
device = 'cpu'
if hasattr(torch, "xpu") and torch.xpu.is_available():
    device = 'xpu'
elif torch.cuda.is_available():
    device = 'cuda'
```

## 3. Запуск и мониторинг (Понимание Loss)

Запустите тренировку через Hub: `python pythagoras_hub.py` -> пункт `1`.

На экране появятся бегущие строки, проценты и цифры. Главный математический показатель, за которым вам нужно пристально следить — это **Loss (Функция потерь)**. В нашем случае используется Cross-Entropy Loss.

> [!IMPORTANT]
> **Loss** — это математическая оценка того, насколько сильно модель "удивляется" или "ошибается" по сравнению с правильным ответом из датасета. **Чем ниже Loss, тем лучше.** Цель обучения — свести Loss как можно ближе к нулю.

**Этапы падения Loss (Жизненный цикл ИИ):**
*   **`Loss ~ 2.5 - 3.0` (Младенец):** Модель только инициализировалась случайным шумом. Она вообще не понимает цифр и выдает случайные символы.
*   **`Loss ~ 1.0` (Первоклассник):** Модель поняла базовый синтаксис. Она выучила, что после знака `=` должны идти цифры, а не знаки минуса или плюса.
*   **`Loss ~ 0.5` (Сложение):** Модель выучила правила простого поразрядного сложения (без переносов).
*   **`Loss < 0.1` (Математик):** Модель уверенно решает сложные примеры, понимает заимствование при вычитании и перенос единицы в старший разряд при сложении.

## 4. Сохранение прогресса и возобновление

Огромное преимущество Трансформеров в том, что модель не нужно обучать за один мучительный присест!

Каждые несколько сотен шагов (этот параметр настраивается переменной `eval_interval` в коде) скрипт делает паузу, проверяет точность на тестовой выборке и **автоматически сохраняет слепок текущего "мозга" модели** в файл `math_model_weights.pth`.

**Как возобновить обучение:**
Если вы прервете скрипт (например, нажав `Ctrl+C` в терминале) и запустите обучение снова на следующий день, Pythagoras автоматически обнаружит файл `math_model_weights.pth`, загрузит все накопленные знания в оперативную память и продолжит обучение ровно с того места (с того же низкого Loss), где вы остановились!

## 5. Глоссарий процесса обучения

| Термин | Подробное значение для Pythagoras |
| :--- | :--- |
| **Epoch (Эпоха)** | Один полный, от начала и до конца, проход скрипта обучения по всему вашему файлу `input_math.txt`. Обычно для хорошего результата нужно несколько десятков эпох. |
| **Batch Size (Размер пакета)** | Сколько примеров (например, 128 или 256) модель "заглатывает" и анализирует за один шаг (шаг обновления весов). Чем больше Batch, тем стабильнее обучение, но тем больше видеопамяти требуется. |
| **Learning Rate / LR (Шаг обучения)** | Скорость обучения (обычно `3e-4`, что равно `0.0003`). Определяет, насколько сильно модель меняет свои нейронные связи при ошибке. Если шаг слишком большой — модель "сойдет с ума" и Loss улетит в бесконечность. Если слишком маленький — будет учиться вечность. |
| **Optimizer (AdamW)** | Алгоритм-компас (Оптимизатор). Он указывает матрицам весов модели математическое направление (градиент), в какую сторону им нужно измениться, чтобы уменьшить Loss на следующем шаге. |
| **Loss (Cross-Entropy)** | Отрицательное логарифмическое правдоподобие. Чем увереннее модель дает правильный ответ (например 99% вероятности для нужной цифры), тем ближе Loss к нулю. |
