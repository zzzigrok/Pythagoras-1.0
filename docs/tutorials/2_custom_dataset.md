## 1. Где живут настройки датасета? (Разбор `prep_math.py`)

Откройте файл `prep_math.py`. Если вы проскроллите вниз, то найдете основную логику генерации, которая скрыта внутри функции `generate_balanced_math()`.

Внутри этой функции вы найдете массив `configs` — это главный "рецепт" нашего датасета.
```python
configs = [
    # Формат: (Минимальное число, Максимальное число, Количество примеров)
    (1, 9, 100000),      # 100к односложных примеров (напр. 5+3, 9-2)
    (10, 99, 150000),    # 150к двузначных (напр. 80-40, 15+23)
    (100, 999, 250000),  # 250к трехзначных (напр. 120+450, 999-100)
    (1, 999, 50000),     # 50к смешанных разрядов (напр. 1+999, 10+500)
]
```

> [!IMPORTANT]
> **Почему примеров разное количество? (Балансировка)**
> Обратите внимание: мы специально генерируем больше сложных трехзначных примеров (250к), чем простых однозначных (100к). Логика переноса разрядов в сотнях (например, `99+1=100`) дается Трансформеру гораздо сложнее, чем зубрежка таблицы сложения до 10. Если не соблюсти этот баланс, модель научится решать простые задачи идеально, но будет проваливаться на длинных числах.

## 2. Как изменить разрядность чисел? (Балансировка)

Допустим, вы хотите вырастить настоящего гиганта и научить модель складывать четырехзначные числа (от 1000 до 9999). Сделать это очень просто — добавьте новую конфигурацию в массив `configs`:

```python
configs = [
    # ... сохраняем старые конфиги для базовых знаний ...
    (1, 9, 100000),
    (10, 99, 150000),
    (100, 999, 250000),
    (1, 999, 50000),
    # --- НОВАЯ СТРОКА ---
    (1000, 9999, 400000), # 400 тысяч хардкорных четырехзначных примеров!
]
```

> [!WARNING]
> **Осторожно с контекстным окном!**
> Если вы увеличиваете разрядность чисел, максимальная длина строки в датасете неизбежно увеличивается. Например, `9999+9999=19998\n` занимает 16 символов. Убедитесь, что параметр `block_size` в архитектуре модели (в файле `pythagoras_hub.py`, обычно по умолчанию установлен на `64`) имеет достаточный запас, чтобы вместить вашу самую длинную сгенерированную строку!

## 3. Как добавить новые математические операции?

По умолчанию Pythagoras версии 1.0 умеет делать только сложение `+` и вычитание `-`. Допустим, мы хотим совершить прорыв и научить его умножению `*`.

**Шаг 1: Добавляем знак в выбор операции.**
Найдите в цикле `for` внутри генератора строку, где скрипт случайно выбирает оператор:
```python
# Было:
op = random.choice(['+', '-'])

# Стало:
op = random.choice(['+', '-', '*']) # Добавили звездочку
```

**Шаг 2: Обучаем генератор (Python) правильно вычислять ответ.**
Модель учится на правильных ответах, поэтому скрипт генерации должен уметь безошибочно вычислять результат умножения перед записью в файл:
```python
if op == '+':
    ans = a + b
elif op == '-':
    ans = a - b
# --- ДОБАВЛЯЕМ ЛОГИКУ УМНОЖЕНИЯ ---
elif op == '*':  
    ans = a * b
```

## 4. Процесс генерации и создания нового словаря

После любых (даже малейших) изменений в `prep_math.py` вам **необходимо заново сгенерировать датасет**.

Откройте терминал и запустите:
```bash
python prep_math.py
```

**Что произойдет под капотом?**
1. Скрипт создаст новый огромный текстовый файл `input_math.txt` (при наших настройках он может весить от 6 до 20 МБ), содержащий сотни тысяч примеров.
2. Скрипт **просканирует** весь созданный файл `input_math.txt` на предмет уникальных символов. Если вы добавили умножение, он найдет символ `*`. Затем скрипт обновит и перезапишет файл словаря `math_vocab.pkl`, присвоив новому символу уникальный числовой ID.

> [!CRITICAL]
> **Критическое предупреждение об архитектуре!**
> При добавлении новых символов (например `*`), размер вашего словаря (`vocab_size`) увеличится. В этот момент **старые веса модели (`math_model_weights.pth`) станут абсолютно несовместимы с новым датасетом!** Слой `LM Head`, который отвечает за предсказание финального символа, был жестко зафиксирован под старый размер словаря. Вам придется удалить старый файл `.pth` и обучать модель полностью с нуля на новых данных.

## 5. Глоссарий Датасета

| Термин | Простое и понятное объяснение |
| :--- | :--- |
| **Dataset (Датасет)** | Набор данных для обучения. В нашем случае это огромный текстовый файл `input_math.txt`, содержащий сотни тысяч примеров и правильных ответов к ним. |
| **Vocab / Vocabulary (Словарь)** | Файл `math_vocab.pkl`. Своего рода алфавит модели. Это список всех уникальных символов, которые модель способна "прочитать" на входе и "написать" на выходе. |
| **Операнд** | Математический термин, обозначающий числа, участвующие в операции. В примере `15 + 2 = 17`, числа 15 и 2 — это операнды. |
| **Балансировка данных** | Процесс искусственного увеличения количества сложных примеров в датасете, чтобы нейросеть уделяла им больше внимания во время обучения. |
